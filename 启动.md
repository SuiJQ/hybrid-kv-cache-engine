# MoeOwner 推理引擎 — 启动指南

## 一分钟上手

### Windows 用户
**双击** `启动.bat` 即可。按菜单提示操作。

### macOS / Linux 用户
打开终端，进入本文件夹，运行：
```bash
bash 启动.sh
```

---

## 交互菜单说明

启动后你会看到 4 个选项：

| 选项 | 做什么 | 适合谁 |
|:----:|:-------|:-------|
| **1** | 先跑基准测试 → 显示成绩 → 启动 API 服务 | ✅ **推荐首次使用** |
| **2** | 只跑基准测试，不启动服务 | 想单独看性能数据 |
| **3** | 跳过测试，直接启动 API 服务 | 已经调好、赶着用 |
| **4** | 查看系统配置和 GPU 信息 | 排查问题 |
| **q** | 退出 | — |

选择后系统会：
1. 自动搜索当前目录的 `.gguf` 模型文件（找不到会让你手动输入）
2. 询问是否启用 **Goose 推测解码**（输入 `y` 启用，推荐）
3. 执行你的选择

---

## 命令行快捷方式（懂一点的同学）

```bash
# 自动：测试 → 启动（适合 CI / 首次部署）
python launch.py --auto

# 跳过测试，直接启动服务
python launch.py --quick

# 带推测解码 + 自定义测试长度
python launch.py --auto --speculative
BENCH_PROMPT_LEN=256 BENCH_GEN_LEN=256 API_PORT=8080 python launch.py --auto
```

## 基准测试报告解读

测试使用固定长度的合成 prompt，输出类似：

```
========================================================
  MoeOwner Benchmark Report
========================================================
  Prompt length:          128 tokens
  Generation length:      128 tokens
  Speculation:         Goose
========================================================
  Time to first token:    125.3 ms        ← 首字延迟，越低越好
  End-to-end time:         4.235 s        ← 总耗时
  Generated tokens:         128
  Decode steps:              64            ← 推测模式会少于 128
  Throughput:              30.2 tok/s     ← 吞吐量，越高越好
  Per-token latency:       33.09 ms/tok
  Spine ratio (EMA):        0.50
  PLD acceptance EMA:       0.35
========================================================
```

如果 `Speculation: Goose` 且 `Decode steps` < `Generated tokens`，说明推测解码生效了。

## 文件说明

| 文件 | 用途 |

| `启动.bat` | Windows 一键入口 |
